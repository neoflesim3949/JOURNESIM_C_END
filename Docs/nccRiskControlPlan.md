## 1. 目的與法規背景

各國電信法規（如台灣 **NCC 國家通訊傳播委員會** 及各地主管機關）對「於當地提供行動通訊/數據服務予當地居民」設有**電信特許/執照**要求。境外 eSIM/SIM 供應商若將**某國之當地方案**銷售給**該國居民、供其於該國境內使用**，可能被視為未經許可提供當地電信服務，構成法規風險。

因此 FLESIM 於 **C 端（消費者前台）** 建置地理限制機制，**主動避免「本地賣本地」**：當偵測到使用者所屬地區與**單一國家在地方案**之涵蓋國家相符時，前台不顯示該方案，從源頭防止此類交易。

> 定位：本平台為**旅客漫遊型 eSIM**（供出國旅客於**目的地**使用），非提供任何國家之**在地居民本地門號/數據服務**。

---

## 2. 風險定義：在地方案 vs 漫遊方案

法規要防範的是「**在地人購買在地方案、於當地使用**」，其核心對象為**單一國家的在地方案**。多國/區域/全球方案本質為**跨境漫遊產品**，即使涵蓋國家中包含使用者母國，其產品性質仍為旅遊漫遊，非提供當地電信服務。

| 方案類型 | 定義 | 風控處理 |
|---|---|---|
| **單一國家方案** | 涵蓋國家 = 1 | 屬**在地方案**；母國相符即受限 |
| **多國 / 區域 / 全球方案** | 涵蓋國家 ≥ 2 | 屬**漫遊產品**；正常銷售（附漫遊用途條款）|

**判定範例**
| 情境 | 判定 |
|---|---|
| 台灣用戶購買**台灣單國**方案於台灣使用 | ⛔ 本地賣本地 → 隱藏 |
| 台灣用戶購買**日本**方案赴日使用 | ✅ 正常漫遊 |
| 歐洲用戶購買**歐洲 33 國區域**方案（含母國）| ✅ 漫遊產品 → 顯示 + 用途條款 |
| 日本用戶購買**日本單國**方案於日本使用 | ⛔ 本地賣本地 → 隱藏 |

**主要風險類型**：無照提供當地電信服務、規避當地電信/稅務監理。

## 3. 使用者地區判定：兩項訊號

系統以下列**兩項獨立訊號**推斷使用者所屬地區，**任一命中即納入**（採聯集，從嚴認定）：

| # | 訊號 | 來源 | 說明 |
|---|---|---|---|
| 1 | **IP 地理位置** | 連線來源 IP → GeoIP 國別 | 反映使用者實際上網所在地 |
| 2 | **註冊電話號碼地區** | 會員註冊手機國碼（如 +886→台灣、+81→日本）| 反映使用者歸屬地 |

> 兩項為**聯集**：只要任一訊號指向地區 R，即認定使用者「涉及地區 R」。此設計從嚴，以降低法規風險。
>
> **說明**：早期版本曾納入「付款幣別」作為第三訊號，惟幣別易受前台計價設定、跨境卡、USD 全球結算等因素失真（誤判率高、且需大量例外），故本版移除幣別訊號，改以 **IP 與註冊電話** 兩項較精準之訊號為準。

## 4. 控管規則

於一般規則之上，另設**嚴格管制國家清單（Strict List）**作為例外強化：對清單內之國家/地區，凡方案涵蓋該國且使用者屬該國者，**一律隱藏（不論單國或多國/區域方案）**——適用於部分主管機關要求「**連含該國之漫遊方案亦不得售予當地居民**」之較嚴格情形。

```
userRegions = { IP 國別 } ∪ { 註冊電話國碼 }
strictList  = 嚴格管制國家清單（可設定）

對每個方案 plan：
   hit = plan.涵蓋國家 ∩ userRegions              // 方案是否涵蓋使用者地區
   若 hit == ∅：
        → 顯示
   否則：
        若 hit ∩ strictList ≠ ∅：
            → 前台【不顯示】                        // ① 嚴格國家：含即擋（不論單國/多國）
        否則 若 plan 涵蓋國家數 == 1：
            → 前台【不顯示】                        // ② 一般國家：僅單國在地方案受限
        否則：
            → 顯示 + 漫遊用途條款                    // ③ 多國/區域漫遊（非嚴格國家）
```

**判定優先序：嚴格清單 ＞ 單國規則 ＞ 多國放行。**

| 使用者地區落在方案涵蓋內 | 該地區屬嚴格清單 | 方案類型 | 結果 |
|:---:|:---:|:---:|:---:|
| 是 | **是** | 單國 或 多國/區域 | ⛔ **隱藏（嚴格）** |
| 是 | 否 | 單國（在地方案）| ⛔ 隱藏 |
| 是 | 否 | 多國/區域（漫遊）| ✅ 顯示 + 條款 |
| 否 | — | — | ✅ 顯示 |

**效果**：使用者在前台**直接看不到**受限方案，從介面層阻斷購買路徑，而非於結帳時才擋。嚴格清單使個別高敏感國家可**升級為連漫遊方案亦封鎖**。

### 漫遊方案用途條款（適用非嚴格之多國方案）
多國/區域/全球方案雖正常顯示，仍以條款釐清產品定位與使用者責任：
- 方案頁標註：**「本方案為出國漫遊用途，適用於前往涵蓋國家/地區旅行時使用」**。
- 結帳時勾選同意：漫遊用途、非供母國在地常態使用。

## 6. 判定精度與備註

- **IP**：以 GeoIP 服務判定國別；VPN/Proxy 可能影響，故搭配電話訊號聯集降低誤判/規避。
- **電話**：以註冊手機國碼判定，最能反映歸屬地，為主要且較穩定之訊號。
- **方案分級**：以「涵蓋國家數」區分單國（在地）／多國（漫遊）；單國才受限，多國一律以漫遊產品處理。
- **邊界情形**：若日後出現「僅 2 國且互為鄰國」等特殊組合而有疑慮，可再視需要調整門檻（現行以「單國才擋」為準）。
